# Update Context

Update the workspace context files with new or revised information.

## Process

1. **Identify what to update**
   Ask user what context needs updating:
   - Brand information (overview, messaging, voice)
   - Audience information (personas, stakeholders)
   - Strategic information (goals, strategy, messages)
   - Competitive information
   - Historical information (past campaigns, benchmarks)

2. **Gather updates**
   - Ask specific questions about what has changed
   - Understand the reason for the change
   - Clarify any ambiguities

3. **Update relevant files**
   - Edit the appropriate context files
   - Maintain consistent formatting
   - Add update timestamps

4. **Review implications**
   - Identify any other files that may need updating
   - Note if changes affect active campaigns or strategies
   - Suggest follow-up actions if needed

## Context File Locations

- `context/client-profile.md` - Core client information
- `context/brand/` - Brand identity and messaging
- `context/audiences/` - Audience profiles and stakeholders
- `context/strategy/` - Strategic documents
- `context/competitive/` - Competitive landscape
- `context/historical/` - Historical performance

## After Updating

1. Summarize what was changed
2. Note any implications for active work
3. Commit changes with descriptive message
